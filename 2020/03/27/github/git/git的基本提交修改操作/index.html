git的提交修改操作

Posted on 2020-03-27 Edited on 2020-08-08 Views:
Symbols count in article: 539 Reading time ≈ 1 mins.

Goal

为了能管理文件的所作出的修改。

Operation

add到stage

  1. 查看stage
    git status查看stage的状态.
  2. 添加到stage
    版本库中有git给你创建的暂存区（stage）和分支（master）。
    git add .的作用就是将工作区的文件添加到stage。
  3. 删除stage
    git restore --staged <file>..." to unstage

commit到master

接下来就是提交。当我们做这一步之后才能被git所记录（log）
`git commit -m “update”

版本回退

  1. 查看版本历史(master)
    git log
    commit 后面的字符就是版本号
    HEAD就是指针

  2. 返回到某个版本(版本号只需写前几个字符)
    git reset --hard <版本号>

这样文件就会被改为到原来被commit的那一次

  1. 要把旧版本push到GitHub要用强制命令
    git push -u origin master -f
    版本回退之后悔
  1. 查看所有的版本
    因为原来的git log已经看不到还没有返回之前的版本号，所以我们要查看所以的版本
    git reflog
    这样就列出了返回之前的版本号.
  2. 继续执行
    git reset --hard <版本号>

这样我们的文件就会被修改到某次历史
