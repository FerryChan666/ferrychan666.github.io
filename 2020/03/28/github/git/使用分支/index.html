使用分支

Created2020-03-28|Updated2022-05-22|github
|Post View:

前言

分支就是一条时间线，它能帮我们记录对仓库的修改操作，我们如果想要在原来分支的基础之上，创作我们的项目，就可以新建另一条分支，它与原来的master分支相同。然后，就切换到新分支进行工作，这样就对原来的分支起到保护作用，因为它不影响master分支。等我们将工作完成后，就可以合并两条分支，两条分支的记录就变成一样。最后，就可以删除用于工作的分支了。

Anyway

分支就是用于记录不同的历史版本的。

Operation

  1. 查看分支
    git branch

  2. 创建分支
    git branch <name>

  3. 将HAND转移到分支
    git checkout <name>
    或者
    git switch <name>

  4. 创建并切换到分支
    git checkout -b <name>
    或者
    git switch -c <name>

  5. 将某分支合并到当前分支
    git merge <name>

  6. 删除分支
    git branch -d <name>
