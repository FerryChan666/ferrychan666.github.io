KMP算法

KMP算法是模式匹配算法的改进，为了避免i的回溯，和避免出现不匹配的字符时，j直接回溯到1。

next[]数组

  • “前缀”指除了最后一个字符以外，一个字符串的全部头部组合；
  • “后缀”指除了第一个字符以外，一个字符串的全部尾部组合。
  • 手工求解Next数组

我们通过比较对应的字符i前的最大前后缀，让后让next[i]=i之前的串中最大共同前后缀 + 1。

首先，我们规定next[1]=0.

然后，next[2]前，只有一个字符A，那么最大公共前后缀为0，所以next[2]=0+1;

next[3]的判断也是一样，next[3]前的字符串的最大公共前后缀为0，所以next[3]=0+1;

next[4]前的字符串的最大公共前后缀为1，所以next[4]=1+1=2;

….

next[]数组的存在是为了在配对时，模式串中避免j直接回溯到1。而是在模式串中第j个字符发生不匹配时，回到next[j]处(这里的位置是：最大匹配前后缀+1），再比较。

因为要找出模式串中所有位置的回溯点，如果前面的配对成功，模式串与主串的字符是一样的，所以，只需研究模式串就够了。

根据已知的next[]来手工求解nextval[]。

首先，规定nextval[0]=0；

其次，判断Pj = or != Pnext[j]。(P后面的是下标)

if(Pj = Pnext[j])

​ nextval[j] = next[j];

else
