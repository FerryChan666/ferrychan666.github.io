Introduction of the tree

树是一种非线性的存储结构。它分为：

  • 顺序存储结构
    • 双亲存储
  • 链式存储结构
    • 孩子存储
    • 孩子兄弟存储

definition

一棵树由节点组成。可以是0个节点——空树。树定义是一种递归思想。一个唯一的根连接不相交的子树，而子树上的节点又能连接不相交的子树。

可以通过平时对真正的树来记忆：

  • 深度（从根开始）：根从上往下的到该节点个数
  • 高度（从节点开始）：节点A从上往下到达叶子，所经路径获得最多的节点个数。
  • 都是针对某个节点的深度，高度

Detail

双亲存储结构

双亲存储仅仅需要一个一维数组。int tree[maxSize]

它的下标代表此节点的位置，数组元素代表双亲位置。

孩子兄弟存储结构

孩子兄弟存储结构实际上就是一个二叉链表。因为二叉链表中指针域意义不同。lchild变成了child指向一个孩子，rchild变成了sibling指向一个兄弟。

tree->binary tree

  • 步骤

  1. 连线

    连上所有的兄弟节点

  2. 剪线

    除了第一个分支，剪掉每一个节点上的其他分支

  3. 旋转

binary tree->tree

  • 步骤

    1. 旋转

    2. 连线

      把节点和父节点都连上

    3. 剪线

      剪掉所有的兄弟节点

forest->binary tree

  • 先来回顾一下概念，森林是由若干颗互不相交的树的集合

因为我们知道孩子兄弟存储结构的根节点是没有兄弟的，所以将树转换成二叉树也是一定没有右孩子的。所以我们又可以利用第一颗二叉树的根的rchild接上第二颗的根，第二颗的根的rchild接上第三颗的根。以此类推，全部合并成一颗二叉树。

  1. 先把森林中的树转换为二叉树
  2. 合并

binary tree->forest

据上，容易得到。从二叉树的根查找rchild右孩子，让rchild指向的节点与根剪掉，再让它成为第二颗树的根，一直到rchild为NULL。然后，再把这一颗颗的二叉树，转换成树。

  1. 拆开。（根据右孩子）
  2. 把每颗二叉树转换成树

树和森林的遍历

树和森林的遍历是差不多的。都是分为先序遍历和后序遍历。

  • 先序遍历：先访问根，再访问子树的根。就是遵循先根再子树的原则
  • 后序遍历：先访问子树，再访问根。遵循先子树再根的原则

（具体手动的遍历方法就是与二叉树的遍历技巧相同）

  • 同时要注意一点，树和森林转换成二叉树之后，它们的先序遍历顺序与二叉树相同，但后序遍历的次序与二叉树的中序遍历相同。
  • 所以可以得出，如果想要遍历树和森林，那么可以把树和森林转换成二叉树，再对二叉树进行遍历。
