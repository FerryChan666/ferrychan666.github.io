Introduction of the binary tree

definition

  • 二叉树是一种有限制的树。
  1. 限制每个节点的分支(子树)个数。（只能是0，1，2)
  2. 子树有左右之分

满二叉树

满二叉树除了叶子，其余节点都有2个子树。并且叶子都存在与最下一层。

完全二叉树

如果有一颗二叉树（深度为k），它的编号（从左往右，从上往下编）和一颗满二叉树（深度也为k）的编号以及所在的位置相同，则这颗二叉树为完全二叉树。

  • 判断技巧：让这颗二叉树与同样深度的满二叉树比较，满的二叉树从右往左，从下至上删除（一定要按顺序）能得到这颗二叉树，那么它就是完全二叉树。

  • 图中，有F则不是完全二叉树，没有F就是。

二叉树的遍历

这种递归的思想，正好是数学上函数的嵌套。f(f(f(p->lchild)))

同时递归有函数出口，它的作用是结束一个函数（当然还有一种方法就是走完这个函数）。这就知道为什么，如果最里面的函数（第3个）没了，那么程序就会回到第二个函数中，继续执行之前还没有执行完的代码（因为递归没执行完）。

  • 快速知道输出序列的技巧（根据上图）

    • 先在每个节点的固定位置画出1，2，3（先，中，后序）
    • 再按那条线的方向走，碰到数字就输出。

  • 重要技巧

    从图中可见，1，2，3这些数字不只是表示先，中，后序。而且，还是线经过该节点的次数。这又表面了每一个节点都要经过三次。

    据此，如果该树按层划分的话。又能推出在到1的时候是进入本层，到3的时候是退出本层。而且，所经之处1，2，3，对应右边程序中的（1）（2）（3）

层序遍历

根为第1层，从左往右，从上往下遍历。

  • 要求：需要一个循环队列，平时是用结构。下面不用结构。数组类型是树节点指针BTNode* queue[maxSize]，这里是有maxSize个指针，因为[]优先级比*高。另外加上rear和front。
  • 主要思想：两个if，如果lchild和rchild存在，则让队列添加它。

非递归深度遍历

因为在前面的递归深度遍历中，系统默认给我们创建了栈。所以，在非递归遍历中，我们需要自己创建栈。

前序遍历

先出栈并访问，再将右孩子和左孩子进栈。

中序遍历

先将左孩子全部进栈，然后再出栈并访问，最后将右孩子进栈。

后序遍历

逆后序遍历就是先序遍历过程中，对左右子树遍历交换顺序的结果，然后我们把逆后序遍历再逆过来，就完成了后序遍历。我们需要两个栈。

线索二叉树

我们的二叉树本来就有很多的空指针。假如一棵二叉树有n个节点，那么就有2n个指针域。其中分支就有n-1

个（分支就是已经利用了的指针）。所以没有被利用的指针有2n-(n-1)=n+1个，这样就造成了大量的浪费。

我们的线索二叉树就是想要把这些指针给利用开来。就是想记录在遍历过程中的顺序（前继和后继），然而前中后遍历的顺序不一样，自然地导致了前后继不一样。那么线索二叉树又分成了：前序，中序，后序线索二叉树。

这样的话，树中的节点就被连了起来，构成了一个双链表。这样就便于插入删除等操作。

而且，还避免了栈的使用。

如何利用好lchild和rchild，不能直接地让它们指向前后继，还要看它们又没有指向左右孩子（是否为空）。
