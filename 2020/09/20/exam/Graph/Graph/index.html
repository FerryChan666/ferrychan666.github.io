Synopsis

基本概念

图是由有穷的节点集合与有穷的边集合构成。

此外，为了使图与树区别开来。我们用顶点(vertex)这个词来代替节点(node)。

  • 图分为有向图和无向图。
  • 完全图：任意两个顶点都有一条边相连。若是无向完全图有n(n-1)/2条边。有向完全图则两个顶点间有2条边相连。所以它的边数为无向完全图的2倍n(n-1)

  • 连通图和连通分量

    针对于无向图，如果图中任何两个顶点，都有其对应的边（一条或多条），与之相连，即有路径。那么该无向图为连通图。

    否则如果是非连通图，其中的极大连通子图称为连通分量(当中连通到不能再连的子图）。

    • 有n个顶点的连通图，至少要有n-1条边才能保持图的连通。那么，如果是少于n-1条边就一定不是连通图，多于n-1条边就一定有环。

    • 连通图生成树

      树的性质0告诉我们，如果是一棵树，那么总边数=总结点数-1。所以，如果图的节点为n，那么如果有n-1条边，并且把所有的顶点都连上（连通图），那么它就构成了树。

  • 强连通图和强连通分量

    针对于有向图，对于每一对顶点vi和vj，都有从vi到vj和vj到vi的路径。那么称该有向图为强连通图。否则，其中极大连通子图为强连通分量。

Description

邻接矩阵

顾名思义，邻接就是相关联的边，判断两个顶点之间是否存在边。（边与边的关系）

它的存在就是用抽象的数据，表达了图的结构。n代表图中有多少个顶点，e代表有多少条边，edges(arc)代表了点与点之间是否存在边。edges[][]这个二维数组就是一个矩阵，这个矩阵（方阵）包含了n*n个元素，那么这正好是点与点之间相连接的所有情况。

  • 无向表中，两顶点中有边即赋值为1（邻接）。无边即为0（不邻接）

    • 邻接矩阵对称（那么就会导致——1条边占2个格）

  • 有向表中，两顶点中赋值为该边的权值。无边即为无穷大

再把这些值放到矩阵（二维数组）中去。矩阵中A[i][j]，i，j分别表示顶点的编号，有向图中即为i->j的边（弧）

存储结构

首先要明确，我们是用数组来创建的多个单链表来表示邻接表，这样一来数组下标就成了每个顶点的位置。所有的结点都是存储在每个单链表的表头。为了联系这些顶点们，我们开创了边结点类型构造一些边。它们的作用是存储边指向结点的数组下标，和next（这个next在真正的图中是所属顶点的旁边的边，在单链表中就是下一个边结点）

DFS深度优先搜索遍历

  • 首先需要一个visit[maxSize];数组作为顶点是否被访问过的标记

  • 主要思想：从某个顶点a开始，查找没有被访问过的邻接节点b，访问并标记。递归进入b，再查找b中没有被访问过的邻接顶点，访问并标记。（不断重复）直到有一个顶点的邻接顶点被全部访问过visit[p->adjvex]==0或根本没有p==NULL。那么就递归出去，即退回到最近被访问的顶点，再查找没有被访问过的邻接顶点。（其实本质就是一个递归过程，函数套函数，之前没有做完的回来再做）

  • 图的遍历和二叉树的遍历雷同。二叉树的遍历也是递归过程：递归进入左孩子，直到lchild为空，就递归出去。完成之前没有完成的事——进入右孩子。（函数套函数，之前没有做完的回来再做）。树就是对于每个节点访问两个分支。而图呢，就是对每个顶点访问多个分支。

BFS广度优先搜索遍历

  • 主要思想：首先访问起始顶点v，再访问与之相邻顶点的w1,w2,w3….，再依次访问w1,w2,w3…的全部相邻顶点。类似于二叉树的层序遍历。
  • 需要一个辅助队列，来记录

DFS, BFS的邻接矩阵和邻接表在计算机中的可视化实现

  • 说明

    无论是邻接表或者邻接矩阵，都能实现DFS, BFS.

    图中，不一定非要限定DFS和BFS，但要了解算法思想。

深度，广度优先搜索生成树

因为BFS算法是层层遍历。如果是无向图或者权值全都相同的有向图，那么就可以通过BFS来知道最短路径问题。如果我们把这个图的在BFS过程中所经的边保留，其余的边删除，那么就生成了一颗广度优先搜索生成树。（深度优先搜索生成树也是一样类似）
