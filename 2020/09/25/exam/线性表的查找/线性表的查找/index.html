线性表的查找

Posted on 2020-09-25 Edited on 2020-09-26 Views:
Symbols count in article: 2.2k Reading time ≈ 2 mins.

Synopsis

查找的定义：就是给定一个值k，从在含有n个记录的表中找出关键字等于k的记录。

线性表查找分为：

  • 顺序查找
  • 折半查找（二分查找）
  • 分块查找
  • 数据类型定义

Description

顺序查找

顺序查找，既适用于顺序存储结构也适用于链式存储结构。

假设元素从ST.R[1]开始存放数据，ST.R[0]闲置不用。我们从后往前找。

这样就会出现一个问题，循环每次都要判断i是否>=1，然后再执行if语句判断是否等于key。我们可以改进这个程序，**就是利用ST.R[0]=key，它的作用就像是监视的哨兵，如果到了ST.R[0]（它必然==key），那么就return 0就表示找不到了。

折半查找(二分查找)

这种方法就不是从头到尾一个个去对比key。而是折取一半（范围变成一半）比较，不成功再折取一半。那么效率也明显提高了。

  • 不过好的方法也会有限制
    • 查找表中元素只能按有序排列
    • 必须是顺序存储结构，不能是链式

查找算法效率的优劣标准ASL

由于查找算法的基本操作是关键字的比较，并且查找次数与待查找的关键字有关。所以，把对范围内所有关键字的比较次数（平均查找长度average search length）作为算法效率的优劣标准。

分块查找

顾名思义，可以把分块查找简单描述为：把线性表分为若干块，再找。这样一来，我们就需要额外建立一个索引表。

可以看出

  • 每一块表中的元素是任意的

  • 但块与块之间要按关键字的大小有序排列

    所以根据二分查找和顺序查找的适用范围，可以知道分块查找，首先对块进行二分查找，对块中元素进行顺序查找。对应的平均查找长度ASL，就是二分查找的平均查找长度+顺序查找的平均查找长度
