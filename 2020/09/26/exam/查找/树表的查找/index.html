树表的查找

Posted on 2020-09-26 Edited on 2020-12-31 Views:
Symbols count in article: 1.9k Reading time ≈ 2 mins.

Synopsis

因为折半查找拥有苛刻的适用条件，既要是顺序表还要有次序。并且如果进行频繁的插入删除操作对于顺序表来说是效率低下的。所以，为了解决可以更有效率地插入删除，我们使用了二叉树作为查找表的组织形式。

  • 二叉排序树
  • 平衡二叉树

Description

二叉排序树（二叉查找树）

二叉排序树实际上就是二叉树的链式存储结构。以此来进行查找。（对排序和查找都有重要作用）

存储结构

定义

二叉排序树可以为空树，或者是具有以下形式的二叉树：

  1. 左子树上的所有节点都小于它的根节点。
  2. 右子树上所有的节点都大于它的根节点。
  3. 且定义是递归的。（它的左右子树也分别为二叉树）
  • 并由该定义可得一个重要性质：中序遍历一颗二叉树可得一个以节点值递增的序列

基本算法

查找关键字

根据定义，可以知道二叉排列树跟折半查找法是要求都是按顺序排列。所以，我们沿用折半查找的算法思想，先让key跟根的key比较，如果相等则return根节点（递归出口）。如果小于根的key，那么就递归进入左子树，再跟左子树的根的key比较。如果大于根的key那么就进入右子树。。。如果一直到最后什么也没有，那么就自然而然地return NULL。所以又可以知道折半查找的判定树（折半查找生成的）就是二叉排序树。但折半查找判定树是唯一的（因为它是靠有序表而建立起来的），二叉排序树是不唯一的（只要满足定义就可以）因为，树不唯一，深度不为唯一。所以，含有n个节点的ASL也不唯一。

插入关键字

插入算法以查找算法为基础的。要插入一个key，先要向下查找，直到遇到节点为空，则创建节点，并且是新增的叶子节点。如果key等于其中某一个节点，则什么也不做。

创建二叉排序树

创建二叉排序树的算法就是使用插入算法，一个一个地从空树中插入节点。

总结：

二叉排序树的插入算法以查找算法为基础，创建算法以插入算法为基础。

AVL平衡二叉树

平衡二叉树是一种特殊的二叉排序树，又称AVL树。为什么要研究这种特殊的二叉排序树呢？当然是因为它作为查找树的时候，查找效率更高。通过，查找算法可以发现，树的高度越低，查找速度越快。如果二叉树是线性的，查找时间复杂度为O(n)，如果是平衡二叉树复杂度为O(log2 n)

建立平衡二叉树

平衡二叉树的建立也类似与二叉排序树的建立，也是一个一个节点地插入到树中。但要实现平衡。实现平衡就是需要左右子树之间的高度不能超过1。为此为根引入了平衡因子，即 平衡因子 = 左子树的高度 - 右子树的高度

那么关键问题就是在像往常一样创建排序二叉树的过程中，如何进行修改？变成平衡二叉树。平衡二叉树的调整需要引入最小不平衡子树。就是在逐步建立二叉树的过程中找到其中根平衡因子绝对值大于1的子树。

而最小不平衡子树也有不同的形态，分为LL，RR，LR，RL型。这几种类型中都是取出顶上的3个节点。最小的变成左子树，最大的变成右子树，排在中间的变为根。
