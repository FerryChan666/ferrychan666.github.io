Preface

Modern computers store and process information represented as two-valued signals. Binary values work better when building machines that store and process information.

How many sizes to store?

  • A machine-level program views memory as a very large array of bytes, referred to as virtual memory. Rather than accessing individual bit in memory, most computers use blocks of 8 bits, or byte, as the smallest addressable unit of memory.

  • Every byte of memory is identified by a unique number, know as its address, and the set of all possible address is known as the virtual address space.

It simply treats each program object as a block of bytes and the program itself as a sequence of bytes.

Hexadecimal Notation

  • A single byte consists of 8 bits.

    In binary notation, its value ranges from 0000000 to 1111111.

    In decimal notation, its value ranges from 0 to 255.

    In hexadecimal notation, its value ranges from 00 to 0xFF.

  • We write bit pattern as base-16 or hexadecimal number

    Why? Because the binary notation is too verbose, while with decimal notion it is tedious to convert to and from bit patterns.

    • Converting between binary and hexadecimal is straightforward. We can convert 0x173A4Cto binary format by expanding each hexadecimal digit. Conversely, given a binary number, we can convert it to hexadecimal by first splitting it into groups of 4 bit each.

Data Sizes

  • word size

    Every computer has a word size, indicating the size of pointer data. The most important parameter determined by word size is the maximum size of the virtual address space. That is, for a machine with w bits word sizes the virtual addresses can range from 0 to 2^w - 1, giving the program access to at most 2^64 bytes. (A 32-bit word size limits the virtual address space to 4GB, this is, just over 4*10^9 bytes)

Where to store?

Addressing and Byte Ordering

  • The different machine/operating system configurations use different conventions for storage allocation.

  • For ordering the byes representing an object, there are two conventions.

    Let’s have a look. Suppose the variable x of type int and at address 0x100 has hexadecimal value of 0x1234567. Some machines choose to store the object in memory ordered from least significant bytes to most, while other machines storm them from most to least.

    • The former convention -where the least significant byte comes first- is referred to as little endian.
    • The least convention -where the most significant byte comes first- is referred to as big endian.

  • A Function uses hexadecimal to represents the bytes of different program objects.

Representing Code

Different machine types use different and incompatible instruction and encodings. Even identical processors running different operating systems have differences in their coding conventions and hence are not binary compatible. So binary code is seldom portable across different combination of machine and operating system.

A fundamental concept of computer systems is that a program is simply a sequence of bytes. The machine has no information about the original source program.

Operating the integer

Expanding the Bit Representation of a number

We can do some operations in bits to expand the word size while retaining the same numeric value.

  1. Expansion of an unsigned number by zero extension

    Define bit vectors u = [u_w-1, u_w-2, … , 0] of width w and u’ = [0, … , 0, u_w-1, u_w-2, … , 0] of width w’, where w’ > w. Then B2U_w(u) = B2U_w’(u’)

this operation is like Logical right shift of bit operation

  1. Expansion of a two’s complement number by signed extension

    Define bit vectors u = [x_w-1, x_w-2, … , 0] of width w and u’ = [x_w-1, … , x_w-1, x_w-2, … , 0] of width w’, where w’ > w. Then B2T_w(u) = B2T_w’(u’)

this operation is like arithmetic right shift of bit operation

  • This is, assuming two Binary values that are not equal, but when we represent them in signed or unsigned they may be equal.

This operation can be characterized as a form of modular arithmetic. Computing a sum modular $2^w$ by discarding any bits with weight greater than 2^(w-1) in the bit-level representation of x+y.

Unsigned negation

Even the unsigned number is nonnegative. But it also has the unsigned negation. Because the unsigned addition uses the modular addition.

Modular addition, the unsigned addition , forms a mathematical structure known as an abelian group. This is, it has an identity element 0, and every element has an additive inverse.

For every value x, there must be some negation, such than x + negation = 0

For x > 0 , consider the value 2^w - x. We see that $(x + (2^w - x)) mod 2^w = 0$. Hence it is the inverse of x under unsigned addition.

Two’s complement addition

Derivation:

Since the Two’s complement addition has the same bit-level representation as unsigned addition. We can characterize the operation Two’s complement addition as conversing its arguments to unsigned, and performing unsigned addition, and then converting back to two’s complement.

Two’s complement negation

When x = TMin, we cannot represent the negation of x, . Consider $TMin + TMin = -2^{w-1} + (-2^{w-1}) = -2^w$. it will cause negative overflow of two’s complement addition. Hence $TMin + TMin = -2^w mod 2^w = 0$. So TMin is its own additive inverse.

Dividing by powers of 2

  • If we want to divide something by $2^k$, we just shift it right by k bit.

  1. Unsigned division by a power of 2

    Performing a logical right shift of bit vector$$[x_{w-1}, x_{w-2}, … , x_0]$$ by k yields the bit vector$$[0, … , 0, x_{w-1}, x_{w-2}, … , w_k]$$

  2. Two’s complement by a power of 2

    Performing a arithmetically right shift of bit vector$$[x_{w-1}, x_{w-2}, … , x_0]$$ by k yields the bit vector$$[x_{w-1}, … , x_{w-1}, x_{w-2}, … , x_k]$$

If we want to round it toward zero when x is negative, we just add a bias.

$$
bias = (1 << k) - 1
$$
So the expression is :

$$
(x + bias) >> k
$$

  • Summary, we use this function to complement rounding toward zero ,no matter if the number is positive or negative.

IEEE Floating-Point Representation
