确保scrapy项目要在根目录上运行，原因是：

Scrapy 项目通常是一个 Python 包，当你在 Scrapy 项目的根目录之外运行代码时，Python 可能无法正确找到 scrapytutorial 这个包，导致 无法解析 scrapytutorial.items。这是因为 Python 的 模块搜索路径 (sys.path) 不包含 Scrapy 项目的根目录。

Case 1

一般情况下，在浏览器可以看到的数据，但是在源代码中没有的，用的都是Ajax技术

那Ajax是如何异步加载的呢？

通过浏览器开发者工具，我们可以在Network中看到这样一个数据包，看一下它的Response正好就是展示在页面上的数据内容。

通过其头部信息Headers，可以看到是通过GET方法，请求URLhttp://exercise.kingname.info/ajax_1_backend后，得到Response得到的。

Case 2

但以上只是最简单的一种Ajax加载方式，绝大部分的Ajax不会向服务器发送请求。（所以，我们在抓不到这个包）

然后，我们尝试分析网站源码：

其中，在js中，我们找到这一段：{"code": "\u884c\u52a8\u4ee3\u53f7\uff1a\u5929\u738b\u76d6\u5730\u864e"}

他类似于JSON格式字符串，把它转成python中的格式打印，正好就是网页中的内容。

像这种情况，就不属于Ajax异步加载，实际上是把JavaScript数据加密了

所以，我们的解决方法就是提取这段加密代码，经过解析即可。

Case 3

当我们发现Ajax请求是POST方法，这个时候就表示需要传递有关的参数才能得到请求内容。

可以通过payload查到传递的参数，secret1, secret2

经解密，Response中code的值正是我们想要的内容。

所以，我们需要得到两个参数的内容才可以，发送POST请求得到内容，而这些内容可能藏在JavaScript中，也可能是通过其他Ajax请求获得。

在网页源码的JavaScript中，我们得到了secret2

在另一个Ajax请求的内容中，得到了secret1

最终，我们得到了页面中的内容
