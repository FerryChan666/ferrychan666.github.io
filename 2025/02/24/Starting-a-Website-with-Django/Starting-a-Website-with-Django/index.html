Starting a Website with Django

Created2025-02-24|Updated2025-02-24
|Post Views:

Interact with users

  1. create basic Django project structure

  2. start our app

  3. from backend to frontend

    Write a method in the views.py (such as, dealing with the data in database and returning the result to frontend), and response something to user.

    However, the backend has to receive the user’s request. So we should create a urls.py in the videos folder and add the URL in it.

    More step, Configure the global URLconf in the mysite project to include the URLconf defined in apps.

    Now we can run the server and request the URL http://localhost:8000/videos/ to see what happen.

    Amazing! As you can see Django accept our users’ requests from URL and response the data to us!

    Additional, we can render our response to html files. Let’s keep reading!

Render a html file

Above, we just used a view method to respond with text. But I want more. For example, an HTML file.

Before we render a HTML file, we have to know the templates and static folder.

Firstly, we create static and template folders in app folder. The static folder stores asset, css, and js files and the template stores html files.

Secondly, one important thing is setting the templates variable in setting.py. We all need to change is the value of 'DIRS' .

Now we can render our HTML file with views method. This mean Django take the request including all user data to templates. Finally, load videos/index.html to respond to users.

Similarly, we can add more views functions and HTML pages to implement additional features.
