Description

邻接矩阵

顾名思义，邻接就是相关联的边，判断两个顶点之间是否存在边。它的存在就是用抽象的数据，表达了图的结构。n代表图中有多少个顶点，e代表有多少条边，edges(arc)代表了点与点之间是否存在边。edges[][]这个二维数组就是一个矩阵，这个矩阵（方阵）包含了n*n个元素，那么这正好是点与点之间相连接的所有情况。

  • 无向表中，两顶点中有边即赋值为1（邻接）。无边即为0（不邻接）
  • 有向表中，两顶点中赋值为该边的权值。无边即为无穷大

再把这些值放到矩阵（二维数组）中去。矩阵中A[i][j]，i，j分别表示顶点的编号，有向图中即为i->j的边（弧）

邻链表

顶点（vex)用编号int存储，

边(arc弧)用指针ArcNode*存储

DFS深度优先搜索遍历

  • 首先需要一个visit[maxSize];数组作为顶点是否被访问过的标记
  • 主要思想：从某个顶点a开始，查找没有被访问过的邻接节点b，访问并标记。递归进入b，再查找b中没有被访问过的邻接顶点，访问并标记。（不断重复）直到有一个顶点的邻接顶点被全部访问过visit[p->adjvex]==0或根本没有p==NULL。那么就递归出去，即退回到最近被访问的顶点，再查找没有被访问过的邻接顶点。（其实本质就是一个递归过程，函数套函数，之前没有做完的回来再做）
  • 图的遍历和二叉树的遍历雷同。二叉树的遍历也是递归过程：递归进入左孩子，直到lchild为空，就递归出去。完成之前没有完成的事——进入右孩子。（函数套函数，之前没有做完的回来再做）。树就是对于每个节点访问两个分支。而图呢，就是对每个顶点访问多个分支。

BFS广度优先搜索遍历

  • 需要一个队列以存储当前顶点的相邻顶点
  • 主要思想：首先访问起始顶点v，再访问与之相邻顶点的w1,w2,w3….，再依次访问w1,w2,w3…的全部相邻顶点。类似于二叉树的层序遍历

  • 深度，广度优先搜索生成树

    因为BFS算法是层层遍历。如果是无向图或者权值全都相同的有向图，那么就可以通过BFS来知道最短路径问题。如果我们把这个图的在BFS过程中所经理的边保留，其余的边删除，那么就生成了一颗广度优先搜索生成树。（深度优先搜索生成树也是一样类似）

非递归深度遍历

因为在前面的递归深度遍历中，系统默认给我们创建了栈。所以，在非递归遍历中，我们需要自己创建栈。

前序遍历

先出栈并访问，再将右孩子和左孩子进栈。

中序遍历

先将左孩子全部进栈，然后再出栈并访问，最后将右孩子进栈。

后序遍历

逆后序遍历就是先序遍历过程中，对左右子树遍历交换顺序的结果，然后我们把逆后序遍历再逆过来，就完成了后序遍历。我们需要两个栈。

线索二叉树

我们的二叉树本来就有很多的空指针。假如一棵二叉树有n个节点，那么就有2n个指针域。其中分支就有n-1

个（分支就是已经利用了的指针）。所以没有被利用的指针有2n-(n-1)=n+1个，这样就造成了大量的浪费。

我们的线索二叉树就是想要把这些指针给利用开来。就是想记录在遍历过程中的顺序（前继和后继），然而前中后遍历的顺序不一样，自然地导致了前后继不一样。那么线索二叉树又分成了：前序，中序，后序线索二叉树。

这样的话，树中的节点就被连了起来，构成了一个双链表。这样就便于插入删除等操作。

而且，还避免了栈的使用。

如何利用好lchild和rchild，不能直接地让它们指向前后继，还要看它们又没有指向左右孩子（是否为空）。

Introduction of the tree

树是一种非线性的存储结构。它分为：

  • 顺序存储结构
    • 双亲存储
  • 链式存储结构
    • 孩子存储
    • 孩子兄弟存储

definition

一棵树由节点组成。可以是0个节点——空树。树定义是一种递归思想。一个唯一的根连接不相交的子树，而子树上的节点又能连接不相交的子树。

可以通过平时对真正的树来记忆：

  • 深度（从根开始）：根从上往下的到该节点个数
  • 高度（从节点开始）：节点A从上往下到达叶子，所经路径获得最多的节点个数。
  • 都是针对某个节点的深度，高度

Detail

它的下标代表此节点的位置，数组元素代表双亲位置。

孩子兄弟存储结构

孩子兄弟存储结构实际上就是一个二叉链表。因为二叉链表中指针域意义不同。lchild变成了child指向一个孩子，rchild变成了sibling指向一个兄弟。

  1. 拆开。（根据右孩子）
  2. 把每颗二叉树转换成树

树和森林的遍历

树和森林的遍历是差不多的。都是分为先序遍历和后序遍历。

  • 先序遍历：先访问根，再访问子树的根。就是遵循先根再子树的原则
  • 后序遍历：先访问子树，再访问根。遵循先子树再根的原则

（具体手动的遍历方法就是与二叉树的遍历技巧相同）

  • 同时要注意一点，树和森林转换成二叉树之后，它们的先序遍历顺序与二叉树相同，但后序遍历的次序与二叉树的中序遍历相同。
  • 所以可以得出，如果想要遍历树和森林，那么可以把树和森林转换成二叉树，再对二叉树进行遍历。

赫夫曼树

特点

  1. 树的带权路径（WPL）最短。因此称最优二叉树
  2. 没有度为1的节点
  3. 权值越大的节点离根越近

Construction

  1. 先把根权值最小两棵树取出来（a, b)，并作为左右子树构造新的二叉树，根c的权值为a和b的权值之和。
  2. 然后再把权根值最小两棵树树删了（a，b），图6-30
  3. 再把根权值最小两棵树取出来（b，c），并作为左右子树构造新的二叉树，根的权值为b，c权值之和。
  4. 现在只剩下两棵树了，再把它们删了，并作为左右子树创造一颗根为两子树根的和的树，这就是赫夫曼树。

循环队列

它们在一个结构体中。一个数组data[maxSize], front rear这两个指针存储数组下标。如果入队和出队的操作都是以下标+1的形式，那么这两个指针就都会往后移动，最终造成假溢出。
注意：当存在元素时，front是指向第一个元素的前一个元素。而rear直接指向最后一个元素。因为，要辨别空队列和满队列的条件，所以要牺牲一个元素（不放数据）这里

两个状态

  • 空队列
    front == rear

  • 满队列
    front == (rear + 1) % maxSize

两个操作

出队和入队都是先移动指针，再取元素

  • front, rear指针的移动

    平时我们在链表中就是直接++，–。但循环队列要换成下面这种格式

    • 一般的队列(FIFO)qu.rear = (qu.rear + 1) % maxSize

    • 两端都能进行插入删除的队列（因为这里下标会左右移动，不要让数组越界要 +maxSize

      Q->front = (Q->front - 1 + maxSize) % maxSize; //+1，-1很随意，其它记住，不要动

占满整个数组的循环队列

用tag作为队列空和不空标志（0空, 1不空）。fron=rear同时作为队列是否为空和满的标志。那么，两个条件同时配合就能判断改队列是否为空，是否为满。就不用像普通的顺序队列，牺牲掉front指向的元素，就能占满整个数组。
