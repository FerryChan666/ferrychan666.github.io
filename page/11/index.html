前言

函数中使用参数是否会改变主调函数中的变量？全局变量在函数如何改变？
示例
====

函数中的全局变量

在块中，重新定义了局部变量ｘ，（因为同名）所以全局变量暂时被覆盖。当跳出块时ｘ变为全局变量。

因为全局变量随处可用，所以可在块中修改x的值。


所以全局变量很容易被修改，一般将固定值设为全局变量，使用const修饰。

函数中的参数。

一般由主调程序将变量传给函数。

  • 例如函数
  1. 此时函数中的int x, int n已经是原始数据的备份，所以无在函数中如何修改，都无法改变主调函数中相应的值。
  2. 而const int arr[]则是一个数组指针。const 的作用是防止修改数组中的值（能修改地址）。如果不加const，就可以在函数中修改它的值（也能修改地址）

指针和多维数组

Posted on 2020-03-16 Edited on 2020-08-08
Symbols count in article: 486 Reading time ≈ 1 mins.

前言

指针和数组的关系，首先要知道一些人为的表示符号。

  • []　和　*　–>　取值　　　　　　／／所以，这两种写法可以替换
  • &　　　　　–>　取地址
  • & * []　同时存在时可以互相抵消

示例

　　地址　　|　　　　　　　　　　　　值　　　　　　　　　　　　　　　　　|
　　ａ　　　|　　　ａ［０］　　|　　ａ［１］　　　|　　　ａ［２］　　　　|

　ａ［０］　|　ａ［０］［０］　|　ａ［０］［１］　|　　ａ［０］［２］　　|
　　　　　　|　　　　１　　　　|　　　　２　　　　|　　　　　３　　　　　|
　ａ［１］　|　ａ［１］［０］　|　ａ［１］［１］　|　　ａ［１］［２］　　|
　　　　　　|　　　　４　　　　|　　　　５　　　　|　　　　　７　　　　　|
　ａ［２］　|　ａ［２］［０］　|　ａ［２］［１］　|　　ａ［２］［２］　　|
　　　　　　|　　　　　８　　　|　　　　９　　　　|　　　　　１０　　　　|

  • 往下取值，往上地址
    | a |
    | a[0] |
    | a[0][0] |

    重要切换，原来一维数组a[x]是值，而二维数组是地址。Anyway,最终的取值才是值。

等价写法

  1. a[2] == * (a + 2) //先在a的地址上 +2 ,然后取值
  2. str = &str[0];
    //str是数组的地址，&str[0]是首元素的地址
    而规定他们两相等
  • 例题a[2][1]
  1. *(a + 2) = a[2] //在 a地址上 +2 ,然后取值
  2. 得＊(＊(a + 2) + 1) // Don’t forget 在二维数组中，(a + 2)还是地址
    // 在地址上 +1 再取值

Keep in mind, +2 还是地址上的 +2，还没有取值。

存储类别

Posted on 2020-03-15 Edited on 2020-08-08
Symbols count in article: 1k Reading time ≈ 1 mins.

前言

5种存储类别

存储类别说明符

  • auto extern static register
    内容
  • 作用域
  • 链接
  • 存储期

存储类别可以是变量，还可以是函数

说明

作用域（变量的使用范围）

程序中可访问标识符的区域

  • 块作用域
    {}内的

    没有{}的块
    for while if

  • 文件作用域
    声明在块以外的区域

    （又分为外部链接，和内部链接。区分它们看有没有static）

  • 函数作用域

  • 函数原型作用域
    int function(int x, int y);仅在此时有用

链接(访问范围)

  • 外部链接
    可以在不同文件中访问他（但访问时要加上extern)
  • 内部链接
    可以在一个翻译单元内访问

    编译器源代码文件和所有的头文件都看成一个包含信息的单独文件，而这个文件就是翻译单元。
    每个翻译单元均对应一个源代码文件和它所包含的文件。

  • 无链接
    具有块作用域，函数作用域，函数原型作用域的变量都是无链接变量

存储期(使用时间)

  • 静态存储期
    在程序执行期间一直存在
  • 动态存储期
    可以主动控制
  • 自动存储期（局部变量）
    在块中，从进入块而产生，从退出块而结束

示例

  • 文件作用域，外部链接，静态存储期
    file_1.c

    file_2.c
    extern int x; //引用定义

  • 文件作用域，内部链接，静态存储期

  • 块作用域，无链接，静态存储期

  • 块作用域，无链接，自由存储期

  • 寄存器变量（与字典变量属性相同）
    {
    register int x; //特点快速
    }

  1. 函数的存储类别也和变量类似
    int function(int); //默认声明为外部函数
    extern int function(int); //引用声明使用外部函数
    static int function(int); //声明为内部函数

补充

  • 自动存储期的变量
    在进入块中分配内存，
    如果不初始化，则是垃圾值

  • 静态存储期的变量
    在开始程序时分配内存，到程序结束，
    如果不初始化，则为0

  • 静态存储期变量只能被常量赋值

  2. 将源码备份(删除git中的yilia)
    git add .
  3. 提交修改给git
    git commit -m "update"
  4. git 堆到 GitHub
    git push
  5. 将已经修改的本地yilia文件粘贴到原处
  6. 再进行一系列的操作（2， 3， 4）

    推送成功

备份博客源文件到Github

Posted on 2020-03-12 Edited on 2020-08-08
Symbols count in article: 1.3k Reading time ≈ 1 mins.

*前言
在部署完个网站后hexo g，我发现了一个问题。
我的Github上面只有一些实实在在的HTML文件。HTML文件却不同于我们平时所写的 md文件，我们不能直接在HTML上面写博客。假如想要换台电脑写，或者某天本地环境挂了导致文件消失。所以，麻烦就来了。

  • 要用到的命令介绍
  1. git push与git pull是一对推送/拉取分支的git命令。
    git pull <远程主机名> <远程分支名>:<本地分支名>
  2. 本地通过 git软件 将信息传递给 GitHub
  • 解决方案
    有一种可行的方案就是将本地的源代码文件托管到Github上面去。
  • 在Github上面创建一个repo
  1. 把名字命名为与你Hexo文件夹同名（不这样做的话，后面的push origin master会出错。）

比如，要取回origin主机的next分支，与本地的master分支合并，需要写成下面这样 -
git pull origin next:master
如果远程分支(next)要与当前分支合并，则冒号后面的部分可以省略。上面命令可以简写为：
git pull origin next

  • 让git关联远程主机一下
  1. 初始化git.在hexo文件的目录下，
    git init
  2. 告诉Git你的远程主机名master和分支名
    git remote add origin git@github.com:ferrychan666/blog.git
  3. 获取远端分支（这里需要SSH密钥生成，操作在后方)
    git pull origin master
    修改.gitignore文件（如果没有请手动创建一个），在里面加入.log 和 public/ 以及.deploy/。因为每次执行hexo generate命令时，上述目录都会被重写更新。因此忽略这两个目录下的文件更新，加快push速度。
    执行命令以下命令，完成Hexo源码在本地的提交。
  • 在git上添加本地修改的文件
  • git把本地文件推到Github
    git push origin master

关于SSH密钥生成

  1. 创建用户 一直回车直到
  2. 打开那个id_rsa.pub路径，复制里面内容
  3. 进入Github上点击New shhKey 粘贴里面的内容到Key
  4. 验证：ssh -T git@github.com 看到
    Hi ***! You’ve successfully authenticated, but GitHub does not provide shell access.
    表示成功了！！

在hexo一连贯的操作

  • deploy
    hexo clean && hexo g -d
  • push
