在虚拟机上clone博客源码并deploy

Posted on 2020-03-13 Edited on 2020-08-08
Symbols count in article: 465 Reading time ≈ 1 mins.

前言

将博客备份到Github上后，随便上了虚拟机来验证源码能否clone下来。所以就有了一系列操作。（这台虚拟机相当于新电脑）
流程
=====

  • 安装 git nodejs hexo
    Git
    nodejs
  • clone源码
  • hexo 服务 npm install hexo --save
  • 验证身份
    至此就能hexo g -d了
  • 关联远程repo
  1. 初始化git.在hexo文件的目录下，
    git init
  2. 告诉Git你的远程主机名master和分支名
    git remote add origin git@github.com:ferrychan666/blog.git
  3. 获取远端分支（这里需要SSH密钥生成)
    git pull origin master
  • 经过一系列的验证和创建后就能备份源码了

SSH密钥

本机生成这种密钥，来与自己的Github的账号绑定，所以一台机一个密钥才能够将源码备份到Github。（即git push origin master）

关于yilia主题的备份问题

Posted on 2020-03-13 Edited on 2020-08-08
Symbols count in article: 375 Reading time ≈ 1 mins.

前言

待我将博客源码备份到Github上后，我又发现了一个问题。themes这个文件夹并没有备份到，这是一个实实在在的坑，这可能是由于yilia themes中有个文件.git关联到了作者的仓库,故而无法提交到本地git仓库中从而无法成功推送到自己的github上，再者这竟然还是一个隐藏文件。所以，我不得不进行相关的删除操作。
解决
====

  1. 将yilia剪切出来，查看隐藏文件，将.git 和 .gitignore 删除
    .git文件夹：代表此目录本身是一个git仓库
    .gitignore：忽略你不想提交到git仓库的内容
  2. 将源码备份(删除git中的yilia)
    git add .
  3. 提交修改给git
    git commit -m "update"
  4. git 堆到 GitHub
    git push
  5. 将已经修改的本地yilia文件粘贴到原处
  6. 再进行一系列的操作（2， 3， 4）

    推送成功

备份博客源文件到Github

Posted on 2020-03-12 Edited on 2021-01-10
Symbols count in article: 1.3k Reading time ≈ 1 mins.

*前言
在部署完个网站后hexo g，我发现了一个问题。
我的Github上面只有一些实实在在的HTML文件。HTML文件却不同于我们平时所写的 md文件，我们不能直接在HTML上面写博客。假如想要换台电脑写，或者某天本地环境挂了导致文件消失。所以，麻烦就来了。

  • 要用到的命令介绍
  1. git push与git pull是一对推送/拉取分支的git命令。
    git pull <远程主机名> <远程分支名>:<本地分支名>
  2. 本地通过 git软件 将信息传递给 GitHub
  • 解决方案
    有一种可行的方案就是将本地的源代码文件托管到Github上面去。
  • 在Github上面创建一个repo
  1. 把名字命名为与你Hexo文件夹同名（不这样做的话，后面的push origin master会出错。）

比如，要取回origin主机的next分支，与本地的master分支合并，需要写成下面这样 -
git pull origin next:master
如果远程分支(next)要与当前分支合并，则冒号后面的部分可以省略。上面命令可以简写为：
git pull origin next

  • 让git关联远程主机一下
  1. 初始化git.在hexo文件的目录下，
    git init
  2. 告诉Git你的远程主机名master和分支名
    git remote add origin git@github.com:ferrychan666/blog.git
  3. 获取远端分支（这里需要SSH密钥生成，操作在后方)
    git pull origin master
    修改.gitignore文件（如果没有请手动创建一个），在里面加入.log 和 public/ 以及.deploy/。因为每次执行hexo generate命令时，上述目录都会被重写更新。因此忽略这两个目录下的文件更新，加快push速度。
    执行命令以下命令，完成Hexo源码在本地的提交。
  • 在git上添加本地修改的文件
  • git把本地文件推到Github
    git push origin master

关于SSH密钥生成

  1. 创建用户 一直回车直到（中间存在passphrase，如果一直按enter，就表示无密码。）

  1. 打开那个id_rsa.pub路径，复制里面内容
  2. 进入Github上点击New shh Key 粘贴里面的内容到Key
  3. 验证：ssh -T git@github.com 看到
    Hi ***! You’ve successfully authenticated, but GitHub does not provide shell access.
    表示成功了！！

在hexo一连贯的操作

  • deploy
    hexo clean && hexo g -d
  • push
