verbs with and without object

After some verbs we typically or always add a completion - a phrase which completes the meaning of the verb - which can be an adverb or preposition phrase.

  1. Some verbs are followed by an object. (There are called transitive verbs)

    • Some verbs which are typically or always followed by a completion are transitive in there most common meanings.

    • Include base…on, compare…with, interest…in, lend … to, prevent…from, regard…as, remind…of,supply…with

  2. Some verbs are not followed by an object. (There are called intransitive verbs)

    • Some verbs which are typically or always followed by a completion are intransitive in there most common meanings.

  3. Some verbs can be both transitive and intransitive.

verb + to infinitive or bare infinitive

verb + (object) + infinitive

  1. After some verbs, we need to include an object before a to-infinitive

  2. After some verbs, we can’t include an object before a to-infinitive.

verb + (object) + bare infinitive

  1. make, feel, notice, observe, overheard, see, watch, have, let.

Notice, however, that in passive sentences with these verbs, we use a to-infinitive

verb + wh-clause

  1. Some verbs can be followed by a clause beginning with a wh-word(how, what, when, where, which, who or why, whether)

    • These verbs can also be followed by wh-word(expect ‘why’) + to-infinitive

  1. Some verbs must have an object before the wh-clause

    • These verbs can also be followed by wh-word + to-infinitive

  1. whether

    Whether can be followed by wh-word + to-infinitive, but if can’t.

verb + two objects

Some verbs are followed by two objects. Usually the first object is a person and the second object is a thing.

  1. We can reverse the order of the objects if we put the for or to before object1. (this is called a prepositional object)

    • we often use this patter if we want to focus particular attention on the object after for/to

    • we also use it if object1 is a lot longer than object2.

  • Sometime we use for + object with verbs such as book, build, buy, catch, choose, cook, fetch, find, make, order, pour, save.

  • Sometime we use to + object with verbs such as award, give, hand, lend, offer, owe, pass, show, teach, tell, throw.

  • With some other verbs we can use either to or for, including bring, leave, pay, play, post, read, send, sing, take, tell, throw.(There is very little difference in meaning.)

Synopsis

排序分为内部排序和外部排序。内部排序是指待排序记录全部放在内存中实现，而外部排序适用于数据量较大的情况，需要使用到外存。

Description

definition

插入类

直接插入

  • 算法思想

    遍历数组，将每一个数都按照从大到小的顺序插入到适当的位置上。（可以划分为一个有序数组和无序数组）

折半插入

跟直接插入排序类似，只不过在查找插入位置时，使用了折半查找。

实现

希尔排序

希尔排序也叫缩小增量排序。它是直接插入排序算法的一个改进。因为，直接插入排序算法在序列基本有序，排序记录个数较少，满足这两个条件的时候，效率较高。所以，希尔排序在这两方面做出改进。

实质上，先通过对排序数组进行分割（根据增量），分成几组，对这几组进行直接插入排序。然后，缩小增量以增加数据量，再进行分割分组，对这几组进行直接插入排序。经几次分组以后，整个序列基本有序

这样一来直到增量为1，即对数组进行直接插入排序。

交换类

冒泡排序

冒泡排序的基本过程：大的关键字就是一块石头，沉底。小的关键字就像气泡浮上来。

  1. 从第一个关键字和第二个关键字开始比较，如果前面的比后面的大，则交换。否则不交换。然后，再将第二个关键字和第三个比较，判断是否要交换。以此类推，直到最后一个。那么最后一个就是数组中的最大元素。这样第一趟冒泡排序就完成了。
  2. 接着我们进行第二趟，与第一趟不同的是：我们不需要比较已经放到最后的那个最大元素。m--
  3. 我们循环的结束条件：本次循环没有元素的交换。使用我们使用了flag来记录该次循环是否交换。

算法分析

  1. 时间复杂度

    可见，冒泡排序根据原始序列来说，是分情况的。比较有序的序列就不用排序这么多趟，同时也不用排那么多次了。

    • 最坏情况：初始序列为逆序。根据算法思想可知，每趟都会把1个最大关键字沉底，所以，最坏情况，最多能进行$n-1$趟排序。(n为关键字个数)

      据算法代码分析，第一趟需要比较$n-1$次，以后每趟递减一次。所以第一趟，有n-1次。第二趟,n-2……

      可以根据等差数列求和，来算出一共有比较多少次。
      $$
      (n-1)*[(n-1)+1]/2 = (n-1)*n/2
      $$

    • 最好情况：只需要1趟，比较n-1次。（不用交换）

快速排序

快速排序一次交换可消除多个逆序。

选择排序

简单选择排序

  1. 首先设想将数组分成一个有序的和一个无序的数组。一开始全都是无序的
  2. 遍历整个数组。在遍历中，再遍历无序的数组r[1…n]，并找到最小元素，下标记为k。如果k!=1，那么r[k]与r[1]交换。
  3. 第二次，遍历无序数组r[2…n]，并找到最小元素，下标记为k。如果k!=2，那么r[k]与r[2]交换
  4. 以此类推，可以得到。第i趟，遍历r[i…n]，找到最小元素下标k。if(k != i)，那么r[k]与r[i]交换。

树表的查找

Posted on 2020-09-26 Edited on 2020-12-31
Symbols count in article: 1.9k Reading time ≈ 2 mins.

Synopsis

因为折半查找拥有苛刻的适用条件，既要是顺序表还要有次序。并且如果进行频繁的插入删除操作对于顺序表来说是效率低下的。所以，为了解决可以更有效率地插入删除，我们使用了二叉树作为查找表的组织形式。

  • 二叉排序树
  • 平衡二叉树

Description

二叉排序树（二叉查找树）

二叉排序树实际上就是二叉树的链式存储结构。以此来进行查找。（对排序和查找都有重要作用）

存储结构

定义

二叉排序树可以为空树，或者是具有以下形式的二叉树：

  1. 左子树上的所有节点都小于它的根节点。
  2. 右子树上所有的节点都大于它的根节点。
  3. 且定义是递归的。（它的左右子树也分别为二叉树）
  • 并由该定义可得一个重要性质：中序遍历一颗二叉树可得一个以节点值递增的序列

基本算法

查找关键字

根据定义，可以知道二叉排列树跟折半查找法是要求都是按顺序排列。所以，我们沿用折半查找的算法思想，先让key跟根的key比较，如果相等则return根节点（递归出口）。如果小于根的key，那么就递归进入左子树，再跟左子树的根的key比较。如果大于根的key那么就进入右子树。。。如果一直到最后什么也没有，那么就自然而然地return NULL。所以又可以知道折半查找的判定树（折半查找生成的）就是二叉排序树。但折半查找判定树是唯一的（因为它是靠有序表而建立起来的），二叉排序树是不唯一的（只要满足定义就可以）因为，树不唯一，深度不为唯一。所以，含有n个节点的ASL也不唯一。

插入关键字

插入算法以查找算法为基础的。要插入一个key，先要向下查找，直到遇到节点为空，则创建节点，并且是新增的叶子节点。如果key等于其中某一个节点，则什么也不做。

创建二叉排序树

创建二叉排序树的算法就是使用插入算法，一个一个地从空树中插入节点。

总结：

二叉排序树的插入算法以查找算法为基础，创建算法以插入算法为基础。

AVL平衡二叉树

平衡二叉树是一种特殊的二叉排序树，又称AVL树。为什么要研究这种特殊的二叉排序树呢？当然是因为它作为查找树的时候，查找效率更高。通过，查找算法可以发现，树的高度越低，查找速度越快。如果二叉树是线性的，查找时间复杂度为O(n)，如果是平衡二叉树复杂度为O(log2 n)

建立平衡二叉树

平衡二叉树的建立也类似与二叉排序树的建立，也是一个一个节点地插入到树中。但要实现平衡。实现平衡就是需要左右子树之间的高度不能超过1。为此为根引入了平衡因子，即 平衡因子 = 左子树的高度 - 右子树的高度

那么关键问题就是在像往常一样创建排序二叉树的过程中，如何进行修改？变成平衡二叉树。平衡二叉树的调整需要引入最小不平衡子树。就是在逐步建立二叉树的过程中找到其中根平衡因子绝对值大于1的子树。

而最小不平衡子树也有不同的形态，分为LL，RR，LR，RL型。这几种类型中都是取出顶上的3个节点。最小的变成左子树，最大的变成右子树，排在中间的变为根。

Synopsis

查找的定义：就是给定一个值k，从在含有n个记录的表中找出关键字等于k的记录。

平均查找长度ASL

n 是表中的记录总数，p是查找第i个记录的概率。c是查找第i个记录要进行比较的次数（长度）

Description

顺序查找

顺序查找，既适用于顺序存储结构也适用于链式存储结构。

假设元素从ST.R[1]开始存放数据，ST.R[0]闲置不用。我们从后往前找。

这样就会出现一个问题，循环每次都要判断i是否>=1，然后再执行if语句判断是否等于key。我们可以改进这个程序，就是利用ST.R[0]=key，它的作用就像是监视的哨兵，如果到了ST.R[0]（它必然==key），那么就return 0就表示找不到了。

平均查找长度ASL

$$
ASL=(1+2+…+n)/n
$$

分子根据等差数列求和公式，$Sn=n(a1+an)/2$

可得
$$
ASL=(1+n)/2
$$

折半查找(二分查找)

这种方法就不是从头到尾一个个去对比key。而是折取一半（范围变成一半）比较，不成功再折取一半。那么效率也明显提高了。

  • 不过好的方法也会有限制

    • 查找表中元素只能按有序排列
    • 必须是顺序存储结构，不能是链式

  • 算法步骤

    1. 设立当前查找区间，设low, high和mid=(low+high)/2
    2. 用key与中间值比较，如果相等则直接返回。
    3. 如果是key>mid，则令low=mid+1。如果是key<mid，则令high=mid-1;
    4. 循环以上步骤，直至查找成功，或者low>high退出循环

查找算法效率的优劣标准ASL

由于查找算法的基本操作是关键字的比较，并且查找次数与待查找的关键字有关。所以，把对范围内所有关键字的比较次数（平均查找长度average search length）作为算法效率的优劣标准。

p为查找第i个记录的概率(一般为1/n)，c为查找第i个记录所要用到的次数

Synopsis

基本概念

图是由有穷的节点集合与有穷的边集合构成。

此外，为了使图与树区别开来。我们用顶点(vertex)这个词来代替节点(node)。

  • 图分为有向图和无向图。
  • 完全图：任意两个顶点都有一条边相连。若是无向完全图有n(n-1)/2条边。有向完全图则两个顶点间有2条边相连。所以它的边数为无向完全图的2倍n(n-1)

  • 连通图和连通分量

    针对于无向图，如果图中任何两个顶点，都有其对应的边（一条或多条），与之相连，即有路径。那么该无向图为连通图。

    否则如果是非连通图，其中的极大连通子图称为连通分量(当中连通到不能再连的子图）。

    • 有n个顶点的连通图，至少要有n-1条边才能保持图的连通。那么，如果是少于n-1条边就一定不是连通图，多于n-1条边就一定有环。

    • 连通图生成树

      树的性质0告诉我们，如果是一棵树，那么总边数=总结点数-1。所以，如果图的节点为n，那么如果有n-1条边，并且把所有的顶点都连上（连通图），那么它就构成了树。

  • 强连通图和强连通分量

    针对于有向图，对于每一对顶点vi和vj，都有从vi到vj和vj到vi的路径。那么称该有向图为强连通图。否则，其中极大连通子图为强连通分量。

邻接矩阵

顾名思义，邻接就是相关联的边，判断两个顶点之间是否存在边。（边与边的关系）

它的存在就是用抽象的数据，表达了图的结构。n代表图中有多少个顶点，e代表有多少条边，edges(arc)代表了点与点之间是否存在边。edges[][]这个二维数组就是一个矩阵，这个矩阵（方阵）包含了n*n个元素，那么这正好是点与点之间相连接的所有情况。

  • 无向表中，两顶点中有边即赋值为1（邻接）。无边即为0（不邻接）

    • 邻接矩阵对称（那么就会导致——1条边占2个格）

  • 有向表中，两顶点中赋值为该边的权值。无边即为无穷大

再把这些值放到矩阵（二维数组）中去。矩阵中A[i][j]，i，j分别表示顶点的编号，有向图中即为i->j的边（弧）

Introduction

先来明确一些重要的二叉树特性，并且这些对考题有重要作用。

二叉树，就是只能有0, 1, 2这三种分支的节点。所以，如果n0, n1, n2表示有0，1，2条分支的结点个数，那么总结点n=n0+n1+n2

  • 性质0：（这条性质对应任何树都适用，不仅仅是二叉树，也适用于三叉树，四叉树。。。）
    $$
    总边数（总分支）=总节点数-1
    $$

That there are n - 1 edges follows from the fact that each edge connects some node to its parent, and every node except the root has one parent.

  • 性质1：2分支节点总是比0分支（叶子）节点少一个。

$$
叶子=二分支节点-1
$$

  • 性质2：二叉树第k层上最多有节点
    $$
    2^{k-1}
    $$

  • 性质3：高度或深度为k的二叉树最多有节点
    $$
    2^k-1
    $$

  • 性质5：函数Catalan()：给n个节点，能构成h(n)种不同的二叉树（因为有左右之分）

    h(n)=C^n_{2n}/(n+1)
    $$

  • 性质6：具有n个节点的二叉树完全二叉树的高度或深度为
    $$

补充栈是实现递归的重要方式

这种递归的思想，正好是数学上函数的嵌套。f(f(f(p->lchild)))

栈就是实现递归的重要工具。但调用内部函数时，就会记忆本函数执行到的位置（将该位置信息压入栈中），再去执行另外的函数。

同时递归有函数出口，它的作用是结束一个函数。这就知道为什么，如果最里面的函数（第3个）没了，那么程序就会回到第二个函数中，继续执行之前还没有执行完的代码（因为递归没执行完）。

这种记忆的方式不正式树的遍历所需要的吗？因为一个节点可能会有两个分支。又因为程序只能一步一步地执行。所以先走左边，并且记住这个位置。回来再走右边。

    从图中可见，1，2，3这些数字不只是表示先，中，后序。而且，还是线经过该节点的次数。这又表面了每一个节点都要经过三次。 第一次经过时，为先序遍历。第二次经过，为中序遍历。第三次经过，为后序遍历。

中序遍历

先将左孩子全部进栈，然后再出栈并访问，最后将右孩子进栈。

后序遍历

逆后序遍历就是先序遍历过程中，对左右子树遍历交换顺序的结果，然后我们把逆后序遍历再逆过来，就完成了后序遍历。我们需要两个栈。

线索二叉树

我们的二叉树本来就有很多的空指针。假如一棵二叉树有n个节点，那么就有2n个指针域。其中分支就有n-1

个（分支就是已经利用了的指针）。所以没有被利用的指针有2n-(n-1)=n+1个，这样就造成了大量的浪费。

实质：线索二叉树就是想要遍历一次二叉树。过程中，检测结点指针域是否为空，若为空，将它们指向遍历时的前继和后继。然而前中后遍历的顺序不一样，自然地导致了前后继不一样。那么线索二叉树又分成了：前序，中序，后序线索二叉树。

这样的话，树中的节点就被连了起来，构成了一个双链表。这样就便于插入删除等操作。而且，还避免了栈的使用。

利用好lchild和rchild，不能直接地让它们指向前后继，还要看它们又没有指向左右孩子（是否为空）。

赫夫曼树

WPL

首先，需要理解3个概念

  1. 路径长度：路径上的分支(边）数。

  2. 树的路径长度：从根到每个节点的分支数目之和。

  3. 结点带权路径长度：从该节点到根的路径长度 与该节点权值的乘积

  4. 搞懂了这些，树的 带权 路径长度（WPL)就清晰明了了，就是所有叶子节点的 带权 路径长度之和

  • 形如下面图6-32的赫夫曼树，
    $$
    WPL = (71)+(52)+(23)+(43)=35
    $$

赫夫曼树特点

  1. 树的带权路径（WPL）最短。因此称最优二叉树

  2. 权值越大的节点离根越近

  3. 没有度为1的节点

  4. n个叶子的赫夫曼树共有2n-1个节点

    需要说明一下这个特点，回归前面的性质一：现在就是分支为1的节点没了，只剩下2分支和0分支的节点了。因为，2分支节点总是比0分支少一个。又因为在二叉树种，
    $$
    节点总数=0分支+1分支+2分支
    $$
    所以，得出结论，假如叶子为n个，2分支为n-1，那么总节点数为2n-1

Construction

  1. 先把权值最小的两个根两取出来（c, d），并作为左右子树构造新的二叉树，新的二叉树的根节点权值为a和b的权值之和。
  2. 然后再把权根值最小两棵树树删了（c, d），图6-30（由4颗树变成了3课树）。
  3. （以此类推）再把根权值最小两棵树取出来（根的权值为5，6），并作为左右子树构造新的二叉树，根的权值为这两根的权值之和。
  4. 现在只剩下两棵树了，并作为左右子树创造一颗根为两子树根的和的树，这就是赫夫曼树。

赫夫曼编码

  • 主要思想：为出现次数较多的字符编以较短的编码。

  • 赫夫曼编码方法

    例如有字符串S=AAABBACCCDEEA

    1. 先统计各字符出现的次数，以字符为根节点，出现次数作为权值。
    2. 将他们构造成赫夫曼树
    3. 将每一个节点的左右分支编号为0，1。
    4. 最终，从根到每个叶子路径上的编号序列即为该字符的编码。

  • 性质

    前缀编码：任一个编码都不是其他任何编码的前缀。

    1. 赫夫曼编码是最优前缀编码

      为什么？根据赫夫曼树的编码方法，根通往任何叶子的路径（一个编码）都不可能是通往其他叶子节点的子路径（其他编码的前缀）。

Introduction

树是一种非线性的存储结构。它分为：

  • 顺序存储结构
    • 双亲存储
  • 链式存储结构
    • 孩子存储
    • 孩子兄弟存储

definition

一棵树由节点组成。可以是0个节点——空树。树定义是一种递归思想。一个唯一的根连接不相交的子树，而子树上的节点又能连接不相交的子树

可以通过平时对真正的树来记忆：

  • 路径(path)长度：从节点n到节点m的边数

    A path from node n1 to nk is defined as a sequence of nodes n1, n2, . . . , nk

    The length of this path is the number of edges on the path, namely k -1.

    Notice that in a tree there is exactly one path from the root to each node.

  • 深度（从根开始）：根到该节点的边数

    For any node ni, the depth of ni is the length of the unique path from the root to ni. Thus, the root is at depth 0.

  • 高度（从节点开始）：节点到叶子的最长路径长度。

    The height of ni is the longest path from ni to a leaf. Thus all leaves are at height 0.

  • As an example, C的深度为1，高度为2。D的深度为1，高度也为1。

  • 树的高度和深度：树中节点的最大层次（图中为4）

  2. 把每颗二叉树转换成树

树和森林的遍历

树和森林的遍历是差不多的。都是分为先序遍历和后序遍历。

  • 先序遍历：先访问根，再访问子树的根。就是遵循先根再子树的原则
  • 后序遍历：先访问子树，再访问根。遵循先子树再根的原则

（具体手动的遍历方法就是与二叉树的遍历技巧相同）

  • 同时要注意一点，树和森林转换成二叉树之后，它们的先序遍历顺序与二叉树相同，但后序遍历的次序与二叉树的中序遍历相同。
  • 所以可以得出，如果想要遍历树和森林，那么可以把树和森林转换成二叉树，再对二叉树进行遍历。
