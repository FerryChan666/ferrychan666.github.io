Synopsis

基本概念

图是由有穷的节点集合与有穷的边集合构成。

此外，为了使图与树区别开来。我们用顶点(vertex)这个词来代替节点(node)。

  • 图分为有向图和无向图。
  • 完全图：任意两个顶点都有一条边相连。若是无向完全图有n(n-1)/2条边。有向完全图则两个顶点间有2条边相连。所以它的边数为无向完全图的2倍n(n-1)

  • 连通图和连通分量

    针对于无向图，如果图中任何两个顶点，都有其对应的边（一条或多条），与之相连，即有路径。那么该无向图为连通图。

    否则如果是非连通图，其中的极大连通子图称为连通分量(当中连通到不能再连的子图）。

    • 有n个顶点的连通图，至少要有n-1条边才能保持图的连通。那么，如果是少于n-1条边就一定不是连通图，多于n-1条边就一定有环。

    • 连通图生成树

      树的性质0告诉我们，如果是一棵树，那么总边数=总结点数-1。所以，如果图的节点为n，那么如果有n-1条边，并且把所有的顶点都连上（连通图），那么它就构成了树。

  • 强连通图和强连通分量

    针对于有向图，对于每一对顶点vi和vj，都有从vi到vj和vj到vi的路径。那么称该有向图为强连通图。否则，其中极大连通子图为强连通分量。

图的存储结构

  • 链式存储结构

Description

邻接矩阵

顾名思义，邻接就是相关联的边，判断两个顶点之间是否存在边。（边与边的关系）

它的存在就是用抽象的数据，表达了图的结构。n代表图中有多少个顶点，e代表有多少条边，edges(arc)代表了点与点之间是否存在边。edges[][]这个二维数组就是一个矩阵，这个矩阵（方阵）包含了n*n个元素，那么这正好是点与点之间相连接的所有情况。

  • 无向表中，两顶点中有边即赋值为1（邻接）。无边即为0（不邻接）

    • 邻接矩阵对称（那么就会导致——1条边占2个格）

  • 有向表中，两顶点中赋值为该边的权值。无边即为无穷大

再把这些值放到矩阵（二维数组）中去。矩阵中A[i][j]，i，j分别表示顶点的编号，有向图中即为i->j的边（弧）

  • 首先需要一个visit[maxSize];数组作为顶点是否被访问过的标记

  • 主要思想：从某个顶点a开始，查找没有被访问过的邻接节点b，访问并标记。递归进入b，再查找b中没有被访问过的邻接顶点，访问并标记。（不断重复）直到有一个顶点的邻接顶点被全部访问过visit[p->adjvex]==0或根本没有p==NULL。那么就递归出去，即退回到最近被访问的顶点，再查找没有被访问过的邻接顶点。（其实本质就是一个递归过程，函数套函数，之前没有做完的回来再做）

Introduction

先来明确一些重要的二叉树特性，并且这些对考题有重要作用。

二叉树，就是只能有0, 1, 2这三种分支的节点。所以，如果n0, n1, n2表示有0，1，2条分支的结点个数，那么总结点n=n0+n1+n2

  • 性质0：（这条性质对应任何树都适用，不仅仅是二叉树，也适用于三叉树，四叉树。。。）
    $$
    总边数（总分支）=总节点数-1
    $$

That there are n - 1 edges follows from the fact that each edge connects some node to its parent, and every node except the root has one parent.

  • 性质1：2分支节点总是比0分支（叶子）节点少一个。

$$
叶子=二分支节点-1
$$

  • 性质2：二叉树第k层上最多有节点
    $$
    2^{k-1}
    $$

  • 性质3：高度或深度为k的二叉树最多有节点
    $$
    2^k-1
    $$

  • 性质5：函数Catalan()：给n个节点，能构成h(n)种不同的二叉树（因为有左右之分）
    $$
    h(n)=C^n_{2n}/(n+1)
    $$

  • 性质6：具有n个节点的二叉树完全二叉树的高度或深度为
    $$
    \lfloor log_2n\rfloor+1
    $$

definition

二叉树是一种有限制的树。

  1. 限制每个节点的分支(子树)个数。（只能是0，1，2)
  2. 子树有左右之分

满二叉树

满二叉树除了叶子，其余节点都有2个子树。并且叶子都存在与最下一层。

完全二叉树

如果有一颗二叉树（深度为k），它的编号（从左往右，从上往下编）和一颗满二叉树（深度也为k）的编号以及所在的位置相同，则这颗二叉树为完全二叉树。

  • 判断技巧：让这颗二叉树与同样深度的满二叉树比较，满的二叉树从右往左，从下至上删除（一定要按顺序）能得到这颗二叉树，那么它就是完全二叉树。

补充栈是实现递归的重要方式

这种递归的思想，正好是数学上函数的嵌套。f(f(f(p->lchild)))

栈就是实现递归的重要工具。但调用内部函数时，就会记忆本函数执行到的位置（将该位置信息压入栈中），再去执行另外的函数。

同时递归有函数出口，它的作用是结束一个函数。这就知道为什么，如果最里面的函数（第3个）没了，那么程序就会回到第二个函数中，继续执行之前还没有执行完的代码（因为递归没执行完）。

这种记忆的方式不正式树的遍历所需要的吗？因为一个节点可能会有两个分支。又因为程序只能一步一步地执行。所以先走左边，并且记住这个位置。回来再走右边。

  • 快速知道输出序列的技巧（根据上图）

    • 先在每个节点的固定位置画出1，2，3（先，中，后序）
    • 再按那条线的方向走，碰到数字就输出。

  • 重要技巧

    从图中可见，1，2，3这些数字不只是表示先，中，后序。而且，还是线经过该节点的次数。这又表面了每一个节点都要经过三次。 第一次经过时，为先序遍历。第二次经过，为中序遍历。第三次经过，为后序遍历。

据此，如果该树按层划分的话。又能推出在到1的时候是进入本层，到3的时候是退出本层。而且，所经之处1，2，3，对应右边程序中的（1）（2）（3）

前序遍历

先出栈并访问，再将右孩子和左孩子进栈。

中序遍历

先将左孩子全部进栈，然后再出栈并访问，最后将右孩子进栈。

后序遍历

逆后序遍历就是先序遍历过程中，对左右子树遍历交换顺序的结果，然后我们把逆后序遍历再逆过来，就完成了后序遍历。我们需要两个栈。

线索二叉树

我们的二叉树本来就有很多的空指针。假如一棵二叉树有n个节点，那么就有2n个指针域。其中分支就有n-1

个（分支就是已经利用了的指针）。所以没有被利用的指针有2n-(n-1)=n+1个，这样就造成了大量的浪费。

实质：线索二叉树就是想要遍历一次二叉树。过程中，检测结点指针域是否为空，若为空，将它们指向遍历时的前继和后继。然而前中后遍历的顺序不一样，自然地导致了前后继不一样。那么线索二叉树又分成了：前序，中序，后序线索二叉树。

这样的话，树中的节点就被连了起来，构成了一个双链表。这样就便于插入删除等操作。而且，还避免了栈的使用。

利用好lchild和rchild，不能直接地让它们指向前后继，还要看它们又没有指向左右孩子（是否为空）。

赫夫曼树

WPL

首先，需要理解3个概念

  1. 路径长度：路径上的分支(边）数。

  2. 树的路径长度：从根到每个节点的分支数目之和。

  3. 结点带权路径长度：从该节点到根的路径长度 与该节点权值的乘积

  4. 搞懂了这些，树的 带权 路径长度（WPL)就清晰明了了，就是所有叶子节点的 带权 路径长度之和

  • 形如下面图6-32的赫夫曼树，
    $$
    WPL = (71)+(52)+(23)+(43)=35
    $$

赫夫曼树特点

  1. 树的带权路径（WPL）最短。因此称最优二叉树

  2. 权值越大的节点离根越近

  3. 没有度为1的节点

  4. n个叶子的赫夫曼树共有2n-1个节点

    需要说明一下这个特点，回归前面的性质一：现在就是分支为1的节点没了，只剩下2分支和0分支的节点了。因为，2分支节点总是比0分支少一个。又因为在二叉树种，
    $$
    节点总数=0分支+1分支+2分支
    $$
    所以，得出结论，假如叶子为n个，2分支为n-1，那么总节点数为2n-1

Construction

  1. 先把权值最小的两个根两取出来（c, d），并作为左右子树构造新的二叉树，新的二叉树的根节点权值为a和b的权值之和。
  2. 然后再把权根值最小两棵树树删了（c, d），图6-30（由4颗树变成了3课树）。
  3. （以此类推）再把根权值最小两棵树取出来（根的权值为5，6），并作为左右子树构造新的二叉树，根的权值为这两根的权值之和。
  4. 现在只剩下两棵树了，并作为左右子树创造一颗根为两子树根的和的树，这就是赫夫曼树。

赫夫曼编码

  • 主要思想：为出现次数较多的字符编以较短的编码。

  • 赫夫曼编码方法

    例如有字符串S=AAABBACCCDEEA

    1. 先统计各字符出现的次数，以字符为根节点，出现次数作为权值。
    2. 将他们构造成赫夫曼树
    3. 将每一个节点的左右分支编号为0，1。
    4. 最终，从根到每个叶子路径上的编号序列即为该字符的编码。

  • 性质

    前缀编码：任一个编码都不是其他任何编码的前缀。

    1. 赫夫曼编码是最优前缀编码

      为什么？根据赫夫曼树的编码方法，根通往任何叶子的路径（一个编码）都不可能是通往其他叶子节点的子路径（其他编码的前缀）。

Introduction

树是一种非线性的存储结构。它分为：

  • 顺序存储结构
    • 双亲存储
  • 链式存储结构
    • 孩子存储
    • 孩子兄弟存储

definition

一棵树由节点组成。可以是0个节点——空树。树定义是一种递归思想。一个唯一的根连接不相交的子树，而子树上的节点又能连接不相交的子树

可以通过平时对真正的树来记忆：

  • 路径(path)长度：从节点n到节点m的边数

    A path from node n1 to nk is defined as a sequence of nodes n1, n2, . . . , nk

    The length of this path is the number of edges on the path, namely k -1.

    Notice that in a tree there is exactly one path from the root to each node.

  • 深度（从根开始）：根到该节点的边数

    For any node ni, the depth of ni is the length of the unique path from the root to ni. Thus, the root is at depth 0.

  • 高度（从节点开始）：节点到叶子的最长路径长度。

    The height of ni is the longest path from ni to a leaf. Thus all leaves are at height 0.

  • As an example, C的深度为1，高度为2。D的深度为1，高度也为1。

  • 树的高度和深度：树中节点的最大层次（图中为4）

树和森林的遍历

树和森林的遍历是差不多的。都是分为先序遍历和后序遍历。

  • 先序遍历：先访问根，再访问子树的根。就是遵循先根再子树的原则
  • 后序遍历：先访问子树，再访问根。遵循先子树再根的原则

（具体手动的遍历方法就是与二叉树的遍历技巧相同）

  • 同时要注意一点，树和森林转换成二叉树之后，它们的先序遍历顺序与二叉树相同，但后序遍历的次序与二叉树的中序遍历相同。
  • 所以可以得出，如果想要遍历树和森林，那么可以把树和森林转换成二叉树，再对二叉树进行遍历。

KMP算法

KMP算法是模式匹配算法的改进，为了避免i的回溯，和避免出现不匹配的字符时，j直接回溯到1。

next[]数组

  • “前缀”指除了最后一个字符以外，一个字符串的全部头部组合；
  • “后缀”指除了第一个字符以外，一个字符串的全部尾部组合。
  • 手工求解Next数组

我们通过比较对应的字符i前的最大前后缀，让后让next[i]=i之前的串中最大共同前后缀 + 1。

首先，我们规定next[1]=0.

然后，next[2]前，只有一个字符A，那么最大公共前后缀为0，所以next[2]=0+1;

next[3]的判断也是一样，next[3]前的字符串的最大公共前后缀为0，所以next[3]=0+1;

next[4]前的字符串的最大公共前后缀为1，所以next[4]=1+1=2;

….

next[]数组的存在是为了在配对时，模式串中避免j直接回溯到1。而是在模式串中第j个字符发生不匹配时，回到next[j]处(这里的位置是：最大匹配前后缀+1），再比较。

因为要找出模式串中所有位置的回溯点，如果前面的配对成功，模式串与主串的字符是一样的，所以，只需研究模式串就够了。

它们在一个结构体中。一个数组data[maxSize], front rear这两个指针存储数组下标。如果入队和出队的操作都是以下标+1的形式，那么这两个指针就都会往后移动，最终造成假溢出。
注意：当存在元素时，front是指向第一个元素的前一个元素。（是不放数据的）而rear直接指向最后一个元素。因为，要辨别空队列和满队列的条件，所以要牺牲一个元素（不放数据）这里

两个条件

  • 空队列
    front == rear

  • 满队列
    front == (rear + 1) % maxSize

两个操作

出队和入队都是先移动指针，再取元素

  • front, rear指针的移动

    平时我们在链表中就是直接++，--。但循环队列要换成下面这种格式

    • 一般的队列(FIFO)qu.rear = (qu.rear + 1) % maxSize

    • 两端都能进行插入删除的队列（因为这里下标会左右移动，不要让数组越界要 +maxSize

      Q->front = (Q->front - 1 + maxSize) % maxSize; //+1，-1很随意，其它记住，不要动

占满整个数组的循环队列

用tag作为队列空和不空标志（0空, 1不空）。fron=rear同时作为队列是否为空和满的标志。那么，两个条件同时配合就能判断改队列是否为空，是否为满。就不用像普通的顺序队列，牺牲掉front指向的元素，就能占满整个数组。
