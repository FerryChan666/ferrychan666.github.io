上下转型

Posted on 2020-03-29 Edited on 2020-08-08
Symbols count in article: 1k Reading time ≈ 1 mins.

前言

为什么要向上转型？

我们直接用继承父类的子类实例化子类对象不就好了吗？这不就可以既能访问子类的对象，也能访问父类对象了吗？如果我们要用被覆盖的父类的方法或者属性，那么用super调用不就好了吗？

介绍

向上转型

  1. 就是将子类对象赋值给父类类型的变量。
  2. 向上转型是为了用父类变量来接受不同的子类对象，调用方法的时候传参父类对象，可以调用子类里不同的重写方法，得到不同的结果。 应用

如果不用向上转型则必须写两个 doSleep 方法，一个传递 Cat 类对象，一个传递 Dog 类对象。这还是两个子类，如果有多个子类就要写很多相同的方法，造成重复。可以看出向上转型更好的体现了类的多态性，增强了程序的间接性以及提高了代码的可扩展性。当需要用到子类特有的方法时可以向下转型，这也就是为什么要向下转型。

向下转型

必须强制转换
使用向下转型，必须先向上转型

使用分支

Posted on 2020-03-28 Edited on 2020-08-08
Symbols count in article: 483 Reading time ≈ 1 mins.

前言

分支就是一条时间线，它能帮我们记录对仓库的修改操作，我们如果想要在原来分支的基础之上，创作我们的项目，就可以新建另一条分支，它与原来的master分支相同。然后，就切换到新分支进行工作，这样就对原来的分支起到保护作用，因为它不影响master分支。等我们将工作完成后，就可以合并两条分支，两条分支的记录就变成一样。最后，就可以删除用于工作的分支了。

Anyway

分支就是用于记录不同的历史版本的。

Operation

  1. 查看分支
    git branch

  2. 创建分支
    git branch <name>

  3. 将HAND转移到分支
    git checkout <name>
    或者
    git switch <name>

  4. 创建并切换到分支
    git checkout -b <name>
    或者
    git switch -c <name>

  5. 将某分支合并到当前分支
    git merge <name>

  6. 删除分支
    git branch -d <name>

Result

  • 当我们push上去

显然，分支是已经做好准备工作了。

使用SSH加密传输到远程repo

Posted on 2020-03-28 Edited on 2020-08-08
Symbols count in article: 338 Reading time ≈ 1 mins.

前言

为了将本地的repo托管到GitHub上的repo，需要创建SSH　key，通过SSH协议进行传输。并且通过git添加远程repo，进而推送到GitHub

Operation

创建SSH key

这个key和本机相联系，保存在你GitHub的账户中。
ssh-keygen -t rsa -C "youremail@example.com"

关联远程repo

  1. 在GitHub上new一个repo。与想要托管的本地repo同名。
  2. cd到本地repo。
  3. 在git上添加远程repo，
    git remote add origin your.ssh.url
    origin就是远程仓库的名字。

推送

git push -u origin master

这样就把本地的master分支关联且推送到远程的master分支

git的提交修改操作

Posted on 2020-03-27 Edited on 2020-08-08
Symbols count in article: 539 Reading time ≈ 1 mins.

Goal

为了能管理文件的所作出的修改。

Operation

add到stage

  1. 查看stage
    git status查看stage的状态.
  2. 添加到stage
    版本库中有git给你创建的暂存区（stage）和分支（master）。
    git add .的作用就是将工作区的文件添加到stage。
  3. 删除stage
    git restore --staged <file>..." to unstage

commit到master

接下来就是提交。当我们做这一步之后才能被git所记录（log）
`git commit -m “update”

版本回退

  1. 查看版本历史(master)
    git log
    commit 后面的字符就是版本号
    HEAD就是指针

  2. 返回到某个版本(版本号只需写前几个字符)
    git reset --hard <版本号>

这样文件就会被改为到原来被commit的那一次

  1. 要把旧版本push到GitHub要用强制命令
    git push -u origin master -f
    版本回退之后悔
  1. 查看所有的版本
    因为原来的git log已经看不到还没有返回之前的版本号，所以我们要查看所以的版本
    git reflog
    这样就列出了返回之前的版本号.
  2. 继续执行
    git reset --hard <版本号>

这样我们的文件就会被修改到某次历史

java面向对象思想(this,return)

Posted on 2020-03-26 Edited on 2020-08-08
Symbols count in article: 871 Reading time ≈ 1 mins.

前言

面向对象的思想在主方法中,主要实现:

  1. 建对象
  2. 给参数
  3. 要什么方法就调用

了解

  • 类的成员变量：
    定义在方法外可以被类方法访问的是成员变量。按照java的编码规范，成员变量一般被定义成private权限。

  • 类的属性：
    类的成员变量中，外界可以通过set方法和get方法对成员变量进行操作，那么这个成员变量就是类的属性。

  • this.这个关键字主要用于引用本类中的成员变量或者方法将局部变量的值传递给成员变量

  • 给成员变量是为了给本类中的方法使用*
    由于有时类中的属性名和方法中的参数名有时相同,所以目的为了区分类的属性和参数两者.
